# M1: 文件夹 + 归档 — 设计文档

**日期**: 2026-04-06
**状态**: 已确认
**里程碑**: M1

## 概述

为备忘录应用添加笔记本（文件夹）和归档功能，提升组织能力。

### 目标用户

混合用途（工作/学习/生活）、几百条规模的备忘录用户。

### 核心价值

- **笔记本**：按主题/项目分组备忘录，告别扁平列表
- **归档**：完成的项目隐藏但不删除，保持主列表清爽

---

## 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 文件夹层级 | 单层笔记本 | 简单够用，配合标签做细分，适合目标规模 |
| 侧边栏布局 | 笔记本和标签分离 | 标签全局可用，可跨笔记本筛选 |
| 归档方式 | 作为特殊笔记本 | 可见可恢复，实现简单 |
| 笔记本排序 | 按更新时间 | 活跃笔记本在前 |
| 新备忘录归属 | 默认进入「未分类」笔记本 | 降低创建门槛 |

## 数据模型

### 新增 notebooks 表

```sql
CREATE TABLE notebooks (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  icon TEXT DEFAULT '📁',
  sort_order INTEGER DEFAULT 0,
  created_at TEXT DEFAULT datetime('now'),
  updated_at TEXT DEFAULT datetime('now')
);
```

### memos 表新增字段

```sql
ALTER TABLE memos ADD COLUMN notebook_id INTEGER REFERENCES notebooks(id);
ALTER TABLE memos ADD COLUMN is_archived INTEGER DEFAULT 0;
```

### 初始数据

系统启动时确保存在「未分类」笔记本（id=1）和「已归档」虚拟笔记本。

## 功能规格

### 笔记本功能

#### 创建笔记本
- 入口：侧边栏底部「+ 新建笔记本」按钮
- 弹窗：输入名称，确认创建
- 约束：名称不能为空，不能重复

#### 重命名笔记本
- 入口：笔记本右键菜单 → 重命名 / 或双击
- 弹窗：修改名称，确认保存
- 特殊约束：「未分类」不可重命名

#### 删除笔记本
- 入口：笔记本右键菜单 → 删除
- 确认：弹窗确认「删除笔记本？其中的备忘录将移至未分类」
- 行为：笔记本内所有备忘录的 notebook_id 设为 1（未分类）
- 特殊约束：「未分类」不可删除

#### 切换笔记本
- 点击侧边栏笔记本名称
- 主列表刷新显示该笔记本内的备忘录
- 「全部笔记」显示所有非归档备忘录

#### 笔记本排序
- 默认按 `updated_at DESC` 排序
- 最近有备忘录更新的笔记本排在前面
- 「全部笔记」始终在顶部
- 「已归档」始终在底部

#### 备忘录数量显示
- 每个笔记本后显示备忘录数量 `(N)`
- 数量不包含已归档备忘录

### 归档功能

#### 归档备忘录
- 入口：备忘录卡片操作菜单 → 归档
- 行为：备忘录 `is_archived = 1`
- 快捷键：无（避免误操作）

#### 查看归档
- 入口：点击侧边栏「已归档」笔记本
- 显示：所有 `is_archived = 1` 的备忘录
- 视觉：已归档列表背景略暗，区分于正常列表

#### 恢复备忘录
- 入口：已归档备忘录操作菜单 → 恢复到...
- 弹窗：选择目标笔记本
- 行为：`is_archived = 0`，`notebook_id = 目标笔记本ID`

#### 搜索行为
- 全局搜索默认不包含已归档备忘录
- 搜索框旁可勾选「包含归档」

### 侧边栏布局

```
┌─────────────────────┐
│ 笔记本              │
│ ├─ 📁 全部笔记 (127)│ ← 始终在顶
│ ├─ 📁 工作 (45)     │
│ ├─ 📁 学习 (32)     │
│ ├─ 📁 生活 (50)     │
│ └─ + 新建笔记本     │
├─────────────────────┤
│ 标签                │
│ [重要] [待办] [想法]│
├─────────────────────┤
│ 📦 已归档 (12)      │ ← 始终在底，半透明
└─────────────────────┘
```

## API 设计

### 笔记本 API

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/notebooks` | 获取笔记本列表（含备忘录计数） |
| POST | `/api/notebooks` | 创建笔记本 |
| PUT | `/api/notebooks/:id` | 更新笔记本（重命名） |
| DELETE | `/api/notebooks/:id` | 删除笔记本 |

#### GET /api/notebooks

响应：
```json
{
  "success": true,
  "data": [
    { "id": 1, "name": "未分类", "icon": "📁", "memo_count": 10, "is_default": true },
    { "id": 2, "name": "工作", "icon": "📁", "memo_count": 45, "is_default": false }
  ]
}
```

#### POST /api/notebooks

请求：
```json
{ "name": "新笔记本", "icon": "📁" }
```

响应：
```json
{ "success": true, "data": { "id": 3, "name": "新笔记本", "icon": "📁" } }
```

### 备忘录 API 变更

#### GET /api/memos

新增查询参数：
- `notebook`: 按笔记本筛选（ID 或 'all'）
- `archived`: 是否包含归档（true/false，默认 false）

#### POST /api/memos

请求体新增字段：
```json
{ "title": "...", "content": "...", "notebook_id": 2 }
```

默认 `notebook_id = 1`（未分类）。

#### POST /api/memos/:id/archive

归档备忘录。

响应：
```json
{ "success": true, "data": { "id": 123, "is_archived": true } }
```

#### POST /api/memos/:id/restore

恢复备忘录。

请求：
```json
{ "notebook_id": 2 }
```

响应：
```json
{ "success": true, "data": { "id": 123, "is_archived": false, "notebook_id": 2 } }
```

## 前端实现

### 新增组件

1. **NotebookList** — 侧边栏笔记本列表
2. **NotebookModal** — 创建/重命名笔记本弹窗
3. **ArchiveView** — 已归档备忘录视图
4. **RestoreModal** — 恢复到目标笔记本弹窗

### 状态变更

```javascript
// 新增状态
let notebooks = [];
let currentNotebook = 'all'; // 'all' | notebook_id | 'archived'

// 筛选条件扩展
currentFilter = {
  search: '',
  tag: '',
  notebook: 'all',
  archived: false,
  favorite: false

```

### 交互细节

1. **创建备忘录时**
   - 弹窗底部新增「笔记本」下拉选择
   - 默认选中当前筛选的笔记本（若在「全部笔记」则默认「未分类」）

2. **备忘录卡片**
   - 操作菜单新增「归档」选项
   - 已归档卡片操作菜单显示「恢复」而非「归档」

3. **侧边栏**
   - 笔记本可右键菜单（重命名、删除）
   - 「已归档」半透明显示，视觉区分

## 迁移策略

1. **数据库迁移**
   - 创建 notebooks 表
   - 给 memos 添加 notebook_id 和 is_archived 字段
   - 插入默认「未分类」笔记本

2. **现有数据迁移**
   - 现有备忘录 notebook_id 设为 1（未分类）
   - is_archived 设为 0

3. **前端兼容**
   - 若 notebooks 表为空，显示「未分类」作为唯一笔记本
   - 筛选逻辑兼容无笔记本状态
