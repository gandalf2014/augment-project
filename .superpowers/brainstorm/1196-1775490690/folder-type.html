文件夹实现方式

决定数据结构和交互复杂度

方案 A：单层笔记本
📁 工作
📁 学习
📁 生活
📁 项目A

每个备忘录只属于一个笔记本，简单直观

方案 B：多层文件夹
📁 工作
📁 项目A
📁 项目B
📁 学习
📁 英语
📁 编程

支持嵌套，更灵活但交互复杂

方案 A 优点

  • 实现简单，数据库改动小
  • UI 清晰，无嵌套导航
  • 适合几百条规模的场景
  • 已有标签可做交叉筛选

方案 A 缺点

  • 不能做「工作→项目A」这种层级
  • 笔记本多时会变扁平

方案 B 优点

  • 层级组织更灵活
  • 符合传统文件夹心智模型

方案 B 缺点

  • UI 需要树形导航/面包屑
  • 移动/删除时要处理子文件夹
  • 开发工作量大
